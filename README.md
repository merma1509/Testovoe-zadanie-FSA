# Техническое задание (БА+СА) аналитик

## Обзор проекта

Данное репозиторий содержит решение тестового задания для позиции Бизнес-аналитик + Системный аналитик. Проект демонстрирует навыки моделирования бизнес-процессов, формулирования требований и проектирования API.

## Структура решения

```bash
ba-sa-test/
 ├── task1-bpmn/           # Задача 1: BPMN 2.0 модель процесса выдачи IT-оборудования
 │    ├── model.bpmn       # Исходник BPMN модели
 │    ├── model.png        # Визуализация процесса
 │    └── questions.md     # Вопросы и противоречия к описанию
 │
 ├── task2-marketplace-feature/  # Задача 2: Требования к функциональности маркетплейса
 │    ├── user_story.md    # User Story с критериями приемки
 │    ├── use_cases.md     # Варианты использования
 │    └── process_diagram.png  # Диаграмма процесса (опционально)
 │
 ├── task3-registration-api/     # Задача 3: REST API для регистрации пользователей
 │    ├── openapi.yaml     # Спецификация API в формате OpenAPI 3.0
 │    ├── algorithm.md     # Алгоритм работы бэкенда
 │    └── examples.md      # Примеры запросов и ответов
 │
 └── README.md              # Этот файл
```

## Использованные инструменты

- **BPMN моделирование**: draw.io (diagrams.net)
- **API документация**: OpenAPI 3.0 спецификация
- **Форматирование**: Markdown

## Ключевые демонстрируемые навыки

1. **Анализ бизнес-процессов**: Моделирование сложных кросс-функциональных процессов
2. **Выявление требований**: Формализация бизнес-требований в технические спецификации
3. **Системное проектирование**: REST API дизайн с учетом безопасности и масштабируемости
4. **Внимание к деталям**: Учет edge cases и сценариев ошибок
5. **Структурированное мышление**: Четкая организация документации и артефактов

## Подготовка к собеседованию

При подготовке к обсуждению решения рекомендуется обратить внимание на:

- **Task 1**: Обоснование выбора пулов/дорожек в BPMN, обработка исключительных ситуаций
- **Task 2**: Полнота вариантов использования, учет нестандартных сценариев
- **Task 3**: Обоснование выбора HTTP статусов, стратегии валидации и безопасности

Все решения готовы к обсуждению и защите на техническом собеседовании.
